# GET /v1/admin/emails/dlq/stats

**Tag:** Admin - Emails  
**Summary:** DLQ statistics  
**Description:** Returns information about the Dead-Letter Queue (message count, consumer count, etc.).  
**Authentication:** Bearer token required (`Authorization: Bearer <accessToken>`).

---

## Parameters

None.

---

## Request Body

None.

---

## Responses

### 200 — DLQ statistics

**Content-Type:** `application/json`

```json
{
  "messageCount": 5,
  "consumerCount": 0
}
```

| Field           | Type   | Description                                      |
|-----------------|--------|--------------------------------------------------|
| `messageCount`  | number | Number of messages currently in the DLQ          |
| `consumerCount` | number | Number of active consumers listening to the DLQ  |

## Frontend Integration Notes

- Called by `adminEmailsFacade.getDlqStats()` via `useAdminDlqStats` query
- Cache with TanStack Query using `adminEmailsKeys.dlqStats()`
- Use `messageCount` to display a badge or alert when the DLQ is non-empty
- Requires `EMAIL_ADMIN_VIEW` permission — gate with `<Can perform={Permission.EMAIL_ADMIN_VIEW}>`
- On `401`, the facade layer triggers the token refresh flow automatically
