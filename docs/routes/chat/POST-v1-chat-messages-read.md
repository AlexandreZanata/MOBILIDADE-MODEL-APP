# POST /v1/chat/messages/read

**Tag:** Chat  
**Summary:** Mark messages as read  
**Description:** Marks one or all messages of a ride as read.  
**Authentication:** Bearer token required (`Authorization: Bearer <accessToken>`).

---

## Parameters

None.

---

## Request Body

**Content-Type:** `application/json`

```json
{
  "rideId": "019ae45a-01ff-b967-b2f7-4c22c46ecd75",
  "messageIds": ["019ae45a-01ff-b967-b2f7-4c22c46ecd75"]
}
```

| Field        | Type     | Required | Description                                                               |
|--------------|----------|----------|---------------------------------------------------------------------------|
| `rideId`     | string   | Yes      | UUID of the ride                                                          |
| `messageIds` | string[] | No       | Array of message UUIDs to mark as read. Omit to mark all messages as read |

## Frontend Integration Notes

- Called by `chatFacade.markAsRead(input)` via `useMarkChatMessagesRead` mutation
- Call after the user opens the chat view to clear the unread counter
- On `onSuccess`, invalidate `chatKeys.unread(rideId)` to refresh the unread count
- On `401`, the facade layer triggers the token refresh flow automatically
