# GET /v1/chat/users/{userId}/online-status

**Tag:** Chat  
**Summary:** User online status  
**Description:** Returns whether a user is online and when they were last seen.  
**Authentication:** Bearer token required (`Authorization: Bearer <accessToken>`).

---

## Parameters

| Name     | Location | Type   | Required | Description        |
|----------|----------|--------|----------|--------------------|
| `userId` | path     | string | Yes      | UUID of the user   |

**Example URL:**

```
GET /v1/chat/users/019ae45a-01ff-b967-b2f7-4c22c46ecd75/online-status
```

---

## Request Body

None.

---

## Responses

> The spec does not define a `200` response body for this endpoint. The response is expected to return an online status object with `isOnline` and `lastSeenAt` fields.

## Frontend Integration Notes

- Called by `chatFacade.getOnlineStatus(userId)` via `useChatUserOnlineStatus(userId)` query
- Cache with TanStack Query using `chatKeys.onlineStatus(userId)` with a short `staleTime`
- On `401`, the facade layer triggers the token refresh flow automatically
