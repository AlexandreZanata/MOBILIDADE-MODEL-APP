# GovMobile — Architecture & Infrastructure

## Overview

GovMobile is a public administration mobile application built with **React Native + Expo**,
strict TypeScript, Redux Toolkit, and Socket.IO. It serves three user roles — Passenger,
Driver, and Admin — with real-time ride flows, chat, GPS tracking, and push notifications
via OneSignal.

```
┌─────────────────────────────────────────────────────┐
│                    Screens / UI                      │
│  (src/screens/*  +  src/components/*)               │
├─────────────────────────────────────────────────────┤
│               Domain Hooks                           │
│  (src/hooks/*  +  screens/*/use*.ts)                │
├─────────────────────────────────────────────────────┤
│                  Redux Store                         │
│  (src/store/slices/*  +  src/store/api/baseApi.ts)  │
├─────────────────────────────────────────────────────┤
│               Facade Layer                           │
│  (src/services/facades/*)                           │
├──────────────────────┬──────────────────────────────┤
│   HTTP (Axios)       │   WebSocket (Socket.IO)       │
│   src/store/api/     │   src/services/websocket/     │
└──────────────────────┴──────────────────────────────┘
```

### Core Principle — Facade Pattern

All backend communication goes through a facade. Screens and hooks never call Axios or
Socket.IO directly. This enables:

- Swapping real implementations for mocks without touching any screen
- Testing hooks and screens with a `FacadeProvider` injecting mocks
- Isolating API contracts in a single place

Decision documented in `docs/decisions/adr-001-facade-pattern.md`.

Reusable hooks that do not belong to a specific screen:

| Hook                      | Responsibility                                                        |

| `useAuthSession`          | Session hydration, token refresh, driver status restoration           |

| `useNotifications`        | OneSignal lifecycle (link/unlink external user ID)                    |
| `useAppLocationBootstrap` | Location permission initialization                                    |
| `useRideContext`          | Active ride context sync via REST                                     |
| `useReverseGeocode`       | Reverse geocoding of coordinates                                      |

## src/services — Service Layer

### facades/

One facade per API domain. Each facade implements an interface (`IXxxFacade`) and returns
`Result<T, FacadeError>` — never throws exceptions.

| Facade               | Domain                                                |
|----------------------|-------------------------------------------------------|
| `AuthFacade`         | Login, logout, token refresh                          |
| `RideFacade`         | Ride CRUD, lifecycle (accept, finish...)              |
| `FleetFacade`        | Drivers, vehicles, operational status                 |
| `RealtimeFacade`     | WebSocket `/dispatch` — connection, events, telemetry |
| `SearchFacade`       | Geocoding, routes, address search                     |
| `CartographyFacade`  | Cartographic data                                     |
| `ChatFacade`         | Chat messages                                         |
| `RatingsFacade`      | Ride ratings                                          |
| `NotificationFacade` | In-app notifications                                  |
| `ServersFacade`      | Public servant data                                   |
| `CallFacade`         | Voice calls                                           |
| `RunFacade`          | Ride simulation (dev/mock)                            |

#### facades/mock/

Mock implementations of each facade for backend-free development and testing.
`RunSimulation.ts` simulates a complete ride lifecycle in real time.

### websocket/

`DispatchWebSocket.ts` — typed Socket.IO client for the `/dispatch` namespace.
Exposes typed callbacks (`onConnected`, `onNewRideAvailable`, etc.) and a
`setTokenRefresher` for automatic JWT renewal on 401.

### network/

- `ReconnectionManager.ts` — exponential backoff, JWT refresh before each attempt,
  `onSessionExpired` callback when refresh returns null
- `NetworkMonitor.ts` — detects online/offline via `expo-network`

### notifications/

`OneSignalService.ts` — abstraction over `react-native-onesignal` for external user ID
link/unlink and push lifecycle management.

### location/

`LocationService.ts` — abstraction over `expo-location` for permissions and position watching.

## Testing Strategy

### Philosophy

Two test types coexist in the project:

1. **Behavior tests (`.test.ts`)** — verify contracts and expected behaviors
2. **POC tests (`.poc.test.ts`)** — Proof of Concept, explore real component behavior
   in specific scenarios, often used to document bugs and their fixes

### Property-Based Testing (PBT)

The project uses `fast-check` for property-based tests. The methodology follows three phases:

- **Bug Condition** (`.bugCondition.test.ts`) — tests that MUST FAIL on unfixed code,
  confirming the bug exists
- **Preservation** (`.preservation.test.ts`) — tests that MUST PASS before and after the fix,
  ensuring correct behaviors do not regress
- **Fix Checking** — the same bug condition tests, now PASSING after the fix

### Coverage

Minimum threshold: **45% of lines** (global).
Reports generated in `coverage/` in `text`, `lcov`, and `html` formats.

### Test Location

Each module has co-located tests in `__tests__/`:

```
src/
├── components/atoms/__tests__/
├── components/molecules/__tests__/
├── hooks/__tests__/
├── models/__tests__/
├── navigation/__tests__/
├── screens/{Domain}/__tests__/
├── services/facades/__tests__/
├── services/network/__tests__/
├── services/notifications/__tests__/
├── store/__tests__/
├── theme/__tests__/
└── i18n/__tests__/
```

### Jest Configuration (`jest.config.js`)

- Preset: `react-native`
- `maxWorkers: 2` — avoids memory pressure on large suites
- `forceExit: true` — prevents hang from open handles (timers, sockets)
- `testTimeout: 15000ms`
- `@components`, `@services`, `@store`, etc. mapped via `moduleNameMapper`
- `transformIgnorePatterns` configured to transpile Expo/RN ESM packages

## Realtime Data Flow

```
Socket.IO Server (/dispatch)
        │
        ▼

        │
        ▼
RealtimeFacadeImpl           ← src/services/facades/RealtimeFacade.ts

        │
        ├──► onConnectionStatusChange() ──► useRealtimeSession
        │                                        │
        │                                        ▼
        │                                   realtimeSlice
        │                                   (connectionStatus)
        │                                        │
        │                                        ▼
        │                               useDriverLocationStream
        │                               (starts telemetry when 'connected')
        │
        └──► onEvent() ──► useDriverRealtime / usePassengerRealtime
                                │
                                ▼
                          rideSlice / authSlice
```

## Reconnection & Resilience

```
Network drop / socket drop
        │
        ▼
NetworkMonitor (expo-network)
        │
        ▼
useNetworkManager
        │
        ▼
ReconnectionManager
  - exponential backoff
  - JWT refresh before each attempt
  - onSessionExpired() if refresh returns null → toast + logout
        │
        ▼
RealtimeFacade.connect()
        │
        ▼
useRideReconnection (3s timer)
  - waits for 'reconnection-complete' from server
  - REST fallback if not received within 3s
  - confirmConnected() → emits 'connected'
```
